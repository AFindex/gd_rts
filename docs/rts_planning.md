# RTS 项目规划（3D 场景 + 2D 单位/建筑）

## 1. 目标定义
本项目目标是实现一个 2.5D 风格的 RTS：
- 地图与空间逻辑使用 3D。
- 单位与建筑视觉表现使用 2D 资源（Sprite3D / AnimatedSprite3D）。
- 支持标准 RTS 核心循环：采集 -> 建造 -> 出兵 -> 战斗 -> 胜负判定。

## 2. 游戏核心循环设计
核心闭环由以下行为组成：
- 资源获取：工人从资源点采集，返回基地转化为玩家资源。
- 产能扩张：消耗资源放置建筑，解锁新单位与功能。
- 部队组织：单位可被框选、多选、编队并接收指令。
- 战斗对抗：部队自动攻击或接收强制攻击命令。
- 终局判定：基地被摧毁或达到其他胜利条件。

## 3. 系统规划

### 3.1 世界与场景系统
- 主场景负责组合所有子系统：世界、单位管理、建筑管理、UI、输入。
- 地图使用 `Node3D` + 地面网格 + 碰撞体。
- 建议长期使用 `NavigationRegion3D` 和可烘焙导航网格，单位通过 `NavigationAgent3D` 移动。

### 3.2 相机与视角系统
- RTS 相机采用俯视倾斜角：
  - WASD / 边缘滚动平移。
  - 滚轮缩放。
  - 可选 Q/E 旋转。
- 相机移动边界要受地图尺寸约束，防止视野离开战场。

### 3.3 输入与命令系统
- 左键：单选 / 框选。
- 右键：上下文指令（移动、攻击、采集、建造）。
- 命令流程建议解耦：
  - 输入系统负责识别玩家意图。
  - 命令系统负责把意图转换为实体命令。
  - 单位/建筑控制器执行命令。

### 3.4 单位系统
- 单位基础属性：生命、移动速度、攻击力、攻击间隔、阵营、体积。
- 行为状态：Idle、Move、Gather、Return、Attack、Dead。
- 群体移动应做“目标点散列”而不是让所有单位叠在同一点。

### 3.5 建筑系统
- 建筑分为：
  - 功能建筑（基地、兵营、资源回收建筑）。
  - 防御建筑（后续可扩展）。
- 建筑放置流程：
  - 进入放置模式 -> 预览投影 -> 合法性检测（碰撞/地形/距离） -> 确认放置 -> 扣资源。
- 产兵采用队列：排队、倒计时、出生点校验。

### 3.6 资源与经济系统
- 至少 1 种主资源（如矿物）。
- 玩家资源池集中管理，所有消耗/增加统一经过经济系统。
- 资源点具备容量与枯竭状态。

### 3.7 战斗系统
- 目标选择：优先最近敌方单位，后续可拓展仇恨权重。
- 伤害结算：命中后扣血，单位死亡触发销毁和引用清理。
- 攻击距离与冷却采用定时器或固定 Tick 逻辑，避免帧率耦合。

### 3.8 UI 系统
- 顶部资源栏：显示当前资源。
- 底部命令栏：根据选中目标动态切换（单位命令、建筑生产）。
- 状态面板：显示选中对象属性。
- 小地图和控制组可以作为后续扩展。

### 3.9 数据驱动设计
- 单位和建筑属性放入独立配置资源（`Resource`）。
- 逻辑读取配置，不将数值硬编码在脚本里。
- 建议建立基础配置结构：
  - `UnitData`: HP、速度、攻击、造价、建造时间、贴图。
  - `BuildingData`: HP、占地、造价、可生产单位。

### 3.10 AI 与对手行为
- 首版可使用规则驱动 AI：
  - 定时补工人。
  - 资源足够则扩张。
  - 达到军力阈值后进攻。
- 后续再增加侦察、分兵、优先级策略。

## 4. 性能与工程约束
- 单位逻辑采用固定频率 Tick（例如 10-20Hz），降低每帧负担。
- 采用对象池管理频繁创建销毁对象（弹体、特效、临时提示）。
- 大量单位时减少实时路径重算，必要时批处理寻路请求。
- 统一实体注册表，避免反复全场景检索节点。

## 6. 当前实现状态（更新于 2026-03-03）
当前项目已从“纯规划”进入“可交互原型”阶段，已落地功能如下：
- 主场景可运行，包含 3D 地面、光照、单位、建筑、资源点。
- RTS 相机支持平移与滚轮缩放，并具备基础边界限制。
- 选择系统已支持单选、框选、Shift 叠加选择。
- 右键上下文命令已支持：移动、工人采集。
- 资源循环已可用：工人采矿 -> 回基地交付 -> 资源数值增长。
- 建筑放置已可用：兵营预览、位置合法性判断、放置扣资源。
- 生产队列已可用：基地排工人、兵营排士兵、定时出生。
- 基础 UI 已可用：资源显示与操作提示文本。

### 6.1 当前可操作键位
- `WASD / 方向键`：移动相机。
- `鼠标滚轮`：缩放相机。
- `左键`：单选；拖拽框选；`Shift + 左键` 叠加选择。
- `右键`：对地移动；对资源点下采集命令（仅工人）。
- `B`：进入兵营放置模式（再次左键确认放置，右键或 `ESC` 取消）。
- `R`：对选中基地排队生产工人（消耗矿物）。
- `T`：对选中兵营排队生产士兵（消耗矿物）。

### 6.2 当前已知限制
- 单位移动仍为直线移动，尚未接入 `NavigationAgent3D`，复杂障碍绕行能力不足。
- 战斗系统尚未落地（攻击、伤害、死亡、目标选择未实现）。
- 建造系统仅支持兵营，尚未加入完整建筑树与科技依赖。
- UI 仍是原型层级，缺少单位详情、命令面板、小地图等。
- 经济模型目前仅有单资源（Minerals），尚无维护费/人口上限等机制。

## 7. 接下来要做的内容
- 接入 `NavigationRegion3D + NavigationAgent3D`，替换当前直线移动逻辑。
- 加入战斗闭环：生命值、攻击范围、攻击冷却、死亡回收。
- 扩展建筑体系：更多建筑类型、前置关系、建造时间与建造进度可视化。
- 将单位与建筑数值迁移到 `Resource` 配置，减少硬编码。
- 增加敌方 AI 与胜负判定，形成单局可结束的最小玩法。
- 完善 UI（资源条、选中面板、生产队列面板）并统一风格。

## 8. 迭代原则
- 先保证“全链路可玩”，再提升深度和表现。
- 每次新增功能必须通过最小可测试场景验证。
- 新系统优先模块化，减少节点之间的硬引用。
- 数据优先配置化，降低后期平衡成本。
